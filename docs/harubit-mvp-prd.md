# ☀️ **하루빛 HARUBIT**

## **1. 서비스 한 줄 정의**

하루빛은 **개인과 공동체가 함께 성경을 읽고, 묵상을 기록하며, 통독 현황을 관리하는 신앙 루틴 플랫폼**입니다.

---

# **2. 왜 이 서비스를 만드는가**

## **해결하려는 문제**

### **1. 개인의 신앙 루틴 지속 어려움**

성경 읽기와 묵상은 의지는 있지만 혼자서는 꾸준히 이어가기 어렵습니다.

### **2. 공동체 통독 관리의 불편함**

교회, 청년부, 소그룹에서 통독을 진행해도 누가 읽었는지, 얼마나 참여했는지 관리하기 어렵습니다.

### **3. 묵상과 나눔의 단절**

말씀은 읽지만 기록이 남지 않고, 공동체 안에서 자연스럽게 나누어지지 않습니다.

---

# **3. 서비스 핵심 방향**

하루빛은 단순한 성경 앱이 아니라 아래 구조를 가진 서비스입니다.

```
말씀 읽기
  ↓
묵상 기록
  ↓
키워드 공유
  ↓
공동체 나눔
  ↓
통독 현황 확인
  ↓
신앙 루틴 지속
```

---

# **4. 타겟 사용자 정의**

## **1차 타겟**

| **사용자** | **설명** | **핵심 니즈** |
| --- | --- | --- |
| 개인 신앙인 | 매일 말씀을 읽고 묵상하고 싶은 사용자 | 오늘 읽을 말씀, 간단한 기록 |
| 청년 / 직장인 | 바쁜 일상 속에서 짧게 신앙 루틴을 만들고 싶은 사용자 | 모바일 친화적 UI, 빠른 기록 |
| 소그룹 리더 | 구성원들의 통독 참여를 관리해야 하는 사용자 | 참여율, 읽음 현황, 그룹 관리 |

## **2차 타겟**

| **사용자** | **설명** | **핵심 니즈** |
| --- | --- | --- |
| 교회 관리자 | 교회/공동체 단위로 통독 프로그램을 운영하는 사람 | 조직 관리, 통계, 코스 관리 |
| 공동체 멤버 | 함께 읽고 서로의 묵상을 나누는 사용자 | 피드, 공감, 댓글, 키워드 공유 |

# **5. 기능 범위**

### **개인 루틴 + 공동체 통독 결합**

- 개인은 매일 말씀을 읽고 기록
- 공동체는 함께 통독하고 현황 확인
- 리더는 참여율 관리

---

# **6. MVP 핵심 기능 리스트**

## **MVP 기준**

**“사용자가 모임에 참여해서 오늘의 성경을 읽고, 읽음 체크와 한 줄 묵상을 남기고, 리더가 참여 현황을 확인할 수 있다.”**

# **12. 최종 STEP 1 산출물**

## **서비스 한 줄 정의**

**하루빛은 개인과 공동체가 함께 성경을 읽고, 묵상을 기록하며, 통독 현황을 관리하는 신앙 루틴 플랫폼입니다.**

---

## **핵심 기능 리스트**

```
1. 회원가입 / 로그인
2. 모임 생성
3. 초대 코드로 모임 참여
4. 기본 통독 코스 제공
5. 오늘의 말씀 보기
6. 읽음 체크
7. 한 줄 묵상 작성
8. 키워드 작성
9. 공동체 피드 보기
10. 리더용 통독 현황 확인
11. 관리자용 통독 코스 관리
```

---

## **사용자 흐름**

```
사용자:
로그인 → 모임 참여 → 오늘 말씀 읽기 → 읽음 체크 → 묵상 작성 → 피드 확인

리더:
로그인 → 모임 생성 → 코스 선택 → 초대 코드 공유 → 참여 현황 확인

관리자:
로그인 → 통독 코스 생성 → 날짜별 말씀 등록 → 전체 모임/사용자 관리
```
